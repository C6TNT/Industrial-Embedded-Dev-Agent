## Boundary

This freeze does not authorize:

- real board access;
- SSH/scp/reboot;
- remoteproc reload or firmware deployment;
- EtherCAT `start-bus` or `stop-bus`;
- `0x86`;
- `0x41F1`;
- takeover;
- IO output;
- robot motion.

Future real-device testing is unaffected. It should start from the frozen
offline baseline, open the correct hardware boundary explicitly, save new
timestamped evidence, and only then decide whether to promote a later baseline.

## 十、补齐 formal merge assistant 与 apply dry-run

在前面的 `finish -> review -> promote -> merge plan` 已经打通之后，今天又把 pending 区到正式数据区之间的最后一层辅助链补上了。

这套输出的意义是：

- 把 case candidate 打包成正式材料候选 bundle
- 把 benchmark candidate 变成 append-ready 的 JSONL 草稿
- 把 material index 的拟新增条目提前整理出来
- 让后续正式 merge 时不需要再从零手工汇总

- `6be4e4e Add formal merge assistant for pending dataset`

- `ieda tools apply-formal-merge --dry-run`

- `data/pending/formal_merge_assistant/apply_formal_merge_dry_run.json`
- `data/pending/formal_merge_assistant/apply_formal_merge_dry_run.md`

它会明确告诉我们：

- 哪些 case candidate 预计并入 `data/materials/`

- 哪些 benchmark candidate 可以考虑追加到 `data/benchmark/benchmark_v1.jsonl`
- `data/materials/material_index_v1.md` 可能需要补哪些条目
